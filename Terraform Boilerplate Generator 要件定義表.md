# **要件定義表：Terraform Boilerplate Generator**

## **1\. プロジェクト概要**

本プロジェクトは、インフラエンジニアおよび開発者がTerraformを利用したインフラ構築・CI/CDパイプライン構築を行う際の「基礎的なボイラープレート（テンプレート）ファイル」を自動生成するWebアプリケーションをゼロから構築するものです。

フロントエンドのスタイリング/構築に「Antigravity」を採用し、最終的な成果物は静的サイトとして **GitHub Pages** にてホスティングおよび一般公開することを目的とします。

## **2\. システム・アーキテクチャ構成**

* **プラットフォーム**: Webブラウザ（PC・タブレット・スマートフォン対応）  
* **ホスティング環境**: GitHub Pages (静的ホスティング)  
* **技術スタック**:  
  * HTML5 / CSS3 / JavaScript (Vanilla JS またはビルドツール不要の構成)  
  * **UI/CSSフレームワーク**: Antigravity  
  * **アイコン**: 外部SVGまたは軽量アイコンライブラリ（Lucide等）  
* **インフラ/デプロイフロー**:  
  * GitHubリポジトリの main ブランチへのPushをトリガーに、GitHub Actions経由でGitHub Pagesへ自動デプロイ。

## **3\. 機能要件 (Functional Requirements)**

利用者の選択操作に応じて、リアルタイムにコードを生成・表示する以下の機能を実装します。

### **3.1. CI/CDプラットフォーム選択機能**

* ユーザーは以下のCI/CDパイプラインから1つを選択できる。  
  * GitHub Actions  
  * GitLab CI  
  * Bitbucket Pipelines  
  * CircleCI  
* 選択状態はUI上で視覚的に判別できること。

### **3.2. クラウドプロバイダー選択機能**

* ユーザーは以下のクラウドプロバイダーから1つを選択できる。  
  * Amazon Web Services (AWS)  
  * Google Cloud (GCP)  
  * Microsoft Azure  
  * Oracle Cloud (OCI)  
  * DigitalOcean  
  * Alibaba Cloud  
* 選択状態はUI上で視覚的に判別できること（ラジオボタン等を想定）。

### **3.3. ファイル動的生成機能**

* プラットフォームとプロバイダーの組み合わせに基づき、以下のファイルを自動生成する。  
  1. .gitignore (共通)  
  2. providers.tf (プロバイダーごとに構成・バックエンド設定例を出力)  
  3. variables.tf (プロバイダーごとの必須変数定義)  
  4. main.tf (プロバイダーごとのサンプルリソース定義)  
  5. CI/CDパイプライン設定ファイル（プラットフォームごとにファイル名および内容を動的変更）  
     * 例: .github/workflows/terraform.yml, .gitlab-ci.yml, bitbucket-pipelines.yml, .circleci/config.yml

### **3.4. ファイルプレビュー・切り替え機能**

* 生成された複数のファイルをタブUI（またはリストUI）で切り替えて閲覧できること。  
* 現在選択されているファイル名（パス）と、その中身（コード）を表示する。

### **3.5. クリップボードへのコピー機能**

* 表示中のファイル内容を「コピー」ボタン一つでクリップボードに保存できること。  
* コピー成功時、ボタンのラベルやアイコンが一時的（例：2秒間）に「Copied\!」等に変化し、ユーザーに成功をフィードバックすること。

## **4\. 非機能要件 (Non-Functional Requirements)**

### **4.1. ユーザビリティ・UI/UX**

* **レスポンシブデザイン**: スマートフォンやタブレットなど、画面サイズの小さなデバイスでもサイドバーとメイン画面が崩れることなく操作できること（モバイル時はサイドバーを上部または折りたたみに配置等）。  
* **操作の即時性**: 選択肢を変更した際、画面の再読み込みなし（非同期・DOM操作）で瞬時に生成コードが更新されること。  
* **コードの可読性**: 生成されたコードは等幅フォント（monospace）で表示し、適度な行間とシンタックスハイライト（可能な場合）を提供すること。

### **4.2. パフォーマンス**

* バックエンドAPIを持たない完全な静的サイトであるため、初期レンダリングやアセットのロードが高速に行われること。

### **4.3. 保守性・拡張性**

* 新たなCI/CDプラットフォームやクラウドプロバイダーを容易に追加できるよう、選択肢のデータ（配列やオブジェクト）とテンプレート生成ロジックを分離した設計にすること。

## **5\. 画面構成要素**

* **ヘッダー (Header)**:  
  * アプリケーション名「Terraform Boilerplate Generator」  
  * 簡単な説明文またはGitHubリポジトリへのリンクアイコン  
* **サイドバー / 操作パネル (Sidebar/Controls)**:  
  * CI/CDパイプラインの選択エリア（ボタン・カードUI）  
  * クラウドプロバイダーの選択エリア（リスト・ラジオUI）  
  * 使い方の簡単な説明（Info Box）  
* **メインエリア (Main Content / Editor Viewer)**:  
  * ファイルタブ（切り替え用）  
  * ファイル名および「Copy Code」ボタンのヘッダーバー  
  * コード表示エリア（スクロール可能）
